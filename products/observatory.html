The problem

Knowledge rots the day after you build it.

Sources change, regulations update, contracts renew — and static pipelines never notice. The Observatory watches sources continuously and turns changes into governed updates, not silent drift.

stale pipelinesunwatched sourcessilent driftmanual re-ingestion
What it produces

A loop, not a batch job.

Watch targets

Scope born from what your Brain must keep current — entities, claims, sources, freshness requirements — plus declared discovery scopes.

Drift events

Potential knowledge changes: subject, new evidence, current claim, confidence — never a direct write.

Re-ingestion triggers

New evidence re-enters through Parser; Canonical compares, versions and adjudicates; the Brain serves the new state.

How it works

Three moves. Nothing hidden.

01

Watch

The Brain defines what must stay current; declared scopes add discovery. Sources are admitted by scored decision.

02

Detect & compare

The world is compared against the known state — drift becomes an evidence event, not an edit.

03

Re-enter the flow

Parser structures the new evidence, Canonical adjudicates and versions it, the Brain serves the new state.

Questions

FAQ

Is the Observatory available?

Rolling out. The observation→task funnel is validated by experiment; the API surface ships behind a pilot flag. Ask for access via a pilot.

Which sources can it watch?

Public and private: web sources, portals, feeds, drives, databases. Each is admitted individually — no source enters without a scored decision.

Does the Observatory edit my Brain?

Never directly. It emits drift events; new evidence re-enters through Parser and is adjudicated in Canonical — with versions, contradictions and history preserved — before the Brain serves the new state.
